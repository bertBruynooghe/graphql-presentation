GraphQL vs. plain REST

  • Standardized: graphql.github.io/graphql-spec/
  • Only GET and POST, but 3 operations:
    • query
    • mutation
    • subscription
  • In-response errors vs. HTML error code
  • Mix and match
  • Less HTTP request cycles
